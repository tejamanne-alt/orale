# Expo HAS CHANGED

Read the exact versioned docs at https://docs.expo.dev/versions/v54.0.0/ before writing any code.

Pinned to SDK 54 deliberately, not the npm "latest" tag (57 as of writing): the public
App Store build of Expo Go lags several SDKs behind `expo`'s npm releases, and only
supports the latest one or two. Before bumping this project's SDK, confirm Expo Go
on the store actually supports the target SDK first.
